Deploy Frontend

Guia de Produção: Frontend em Docker

Processo de build autônomo multi-estágio e distribuição estática otimizada via Nginx.

Como Funciona o Build

O deploy de produção do frontend (Vite + React + TypeScript + TailwindCSS) utiliza o padrão Docker Multi-stage. Esse padrão divide o empacotamento em dois momentos bem definidos:

  • Estágio 1 (Builder): Usa uma imagem completa com Node.js para instalar as dependências de desenvolvimento, compilar as fontes TSX e otimizar os assets, gerando um build puramente estático em HTML/JS na pasta dist.
  • Estágio 2 (Runtime): Copia apenas os arquivos gerados em dist para uma imagem pura de Nginx Alpine, descartando o Node e os módulos instalados, resultando em uma imagem final extremamente leve e segura de apenas ~45MB.
Vantagem da Abordagem Autônoma

O host do Homelab não precisa ter o Node instalado nem rodar comandos locais de compilação. Todo o ciclo ocorre de forma isolada dentro do Docker durante a execução do comando compose, garantindo reprodutibilidade idêntica.

Arquivos de Infraestrutura Criados

Abaixo estão os arquivos específicos criados na pasta /app/isalab-os/frontend para operacionalizar esse fluxo:

1. Dockerfile do Frontend

Define as etapas de compilação com Node 20 e distribuição com Nginx.

frontend/Dockerfile 
# Stage 1: Build da SPA
FROM node:20-alpine AS builder

WORKDIR /app

# Copiar arquivos de manifesto de dependências
COPY package.json package-lock.json ./

# Instalar dependências (incluindo devDependencies necessárias para o build)
RUN npm ci

# Copiar código fonte e configurações do projeto
COPY . .

# Compilar o build estático de produção do React/Vite
RUN npm run build

# Stage 2: Runtime com Nginx Alpine leve
FROM nginx:alpine AS runtime

# Copiar os binários compilados (dist) do estágio de build para o diretório padrão do Nginx
COPY --from=builder /app/dist /usr/share/nginx/html

# Copiar a configuração customizada do Nginx contendo a regra de proxy reverso da API
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]

2. O arquivo .dockerignore

Evita o envio de arquivos de build do host para dentro do container, acelerando a inicialização do build.

frontend/.dockerignore 
node_modules
dist
.git
.gitignore
.env
.env.example
README.md
FRONTEND_GUIDE.md
GUIA_FRONTEND.html

3. Nginx do Frontend (nginx.conf)

Configura o roteamento de páginas do SPA (React Router) e o túnel de comunicação local para a API.

frontend/nginx.conf 
server {
    listen 80;
    server_name localhost;

    location / {
        root /usr/share/nginx/html;
        index index.html index.htm;
        try_files $uri $uri/ /index.html;
    }

    # Proxy para o backend Rust
    location /api/ {
        proxy_pass http://homelab-backend:3001/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;

        # Configurações de timeout para operações longas (ex: pipelines)
        proxy_connect_timeout 300s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }
}

Integração com o Docker Compose

O serviço do frontend é integrado no docker-compose.yml apontando para a pasta local como contexto de compilação:

Trecho do docker-compose.yml
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    image: homelab-frontend:latest
    container_name: homelab-frontend
    networks:
      - proxy-network
    restart: always
Roteamento Invisível

Nenhuma porta do frontend é exposta no host local (ex: 80 ou 3000). O Nginx Proxy Manager (NPM), que também roda na rede proxy-network, busca o frontend apontando diretamente para o domínio de container interno: http://homelab-frontend:80.

Comandos de Operação Rápidos

Instruções rápidas para manipulação e reconstrução do container do frontend:

Terminal
# Reconstruir apenas a imagem do frontend e reiniciar o container
docker compose up -d --build frontend

# Verificar se o container está no ar de forma saudável
docker ps --filter "name=homelab-frontend"

# Ler logs do servidor Nginx do frontend
docker logs -f homelab-frontend
